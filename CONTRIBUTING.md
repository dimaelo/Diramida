## Submission Guidelines

### Pull Request (PR) Template

Use this format for PRs:

```markdown
## Description
Brief explanation of what you've changed and why. 

## Type of Change
- [ ] Documentation improvement
- [ ] New feature or research
- [ ] Bug fix
- [ ] Code refactor
- [ ] Content or creative addition

## Why This Change? 
Explain the motivation and what problem it solves.

## Related Issues
Closes #[issue number] or Relates to #[issue number]

## Checklist
- [ ] I've read CONTRIBUTING.md
- [ ] My changes preserve original intent
- [ ] Documentation is updated if needed
- [ ] Commit messages are clear
- [ ] I've tested my changes (if applicable)
```

### Issue Report Template

When opening an issue, include:

1. **Title:** Clear, descriptive heading
2. **Category:** Bug, Enhancement, Discussion, Documentation, Question
3. **Description:** What is the issue or proposal?
4. **Context:** Why is this important?
5. **Suggested Solution:** (Optional) Your idea for addressing it
