## Code of Conduct

This project is committed to fostering a **welcoming, inclusive, and respectful community**. All contributors are expected to: 

- **Be respectful** of different views and backgrounds
- **Assume good intent** in discussions
- **Focus on ideas**, not personal criticism
- **Be patient** with newcomers and different perspectives
- **Report inappropriate behavior** to maintainers

See [CODE_OF_CONDUCT. md](CODE_OF_CONDUCT.md) for full details.

## Getting Started

### 1. Explore the Project
- Read the [README.md](README.md) for overview
- Check [docs/PROJECT_OVERVIEW.md](docs/PROJECT_OVERVIEW.md) for details
- Review [ROADMAP.md](ROADMAP.md) for current priorities
- Join [Discord Community](https://discord.gg/uDyNsfDn) for live discussion

### 2. Find Your Area of Interest
Choose from: 
- üìñ **Documentation** (lowest barrier to entry)
- üî¨ **Research & Ideas** (conceptual work)
- üíª **Technical** (code and tools)
- üé® **Creative** (visual and media)

### 3. Fork or Branch
- For small fixes: Create a branch directly
- For major work: Fork the repository
- Follow naming:  `feature/description` or `docs/improvement`

### 4. Make Your Changes
- Keep commits clear and focused
- Include rationale in commit messages
- Reference issues when applicable
- Preserve original intent ‚Äî improve, don't replace

### 5. Submit & Discuss
- Open a Pull Request (PR) with clear description
- Engage in feedback constructively
- Be open to suggestions and iterations

## Submission Guidelines

### Pull Request (PR) Template

When submitting a PR, use this format:

```markdown
## Description
Brief explanation of changes. 

## Type of Change
- [ ] Documentation improvement
- [ ] New feature or research
- [ ] Bug fix
- [ ] Content/creative addition
- [ ] Infrastructure/tooling

## Why This Change? 
Explain the motivation and problem solved.

## Related Issues
Closes #[issue number]

## Checklist
- [ ] I've read the contributing guidelines
- [ ] My changes preserve original intent
- [ ] Documentation is updated if needed
- [ ] Commit messages are clear and focused
